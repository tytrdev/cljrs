cljrs.js

Browser-side interop, exposed under the js short alias and the qualified cljrs.js namespace (use whichever you prefer; they bind the same builtins). wasm-only — native cljrs builds never see these symbols. Implementation lives in crates/cljrs-wasm/src/js_bridge.rs.

DOM elements round-trip through cljrs as Value::Opaque{tag = "js/element"} so they can be stored in a map or atom and passed back to other builtins. Async results (fetch-text) take a callback that's invoked when the underlying Promise resolves; we can't park a cljrs computation on a JS Promise, so the callback pattern is the integration point.

Console & time

  • (js/log & xs) — console.log; pretty-prints non-strings.
  • (js/now) — Date.now() as f64 ms-since-epoch.

DOM

  • (js/get-element id) — document.getElementById; returns a js/element opaque or nil.
  • (js/set-text! el s) — write textContent; non-string args go through pr-str.
  • (js/set-html! el s) — write innerHTML; string only. (Trusted source — you own the markup.)
  • (js/get-value el) — read .value via Reflect; works for input/textarea/select.
  • (js/set-value! el v) — write .value.

Events

  • (js/on! el event handler) — addEventListener; handler is a 0-arg cljrs fn. The closure leaks (one per listener) — fine for "wire up once at startup", not for per-frame attach/detach.

Storage

  • (js/local-get k) — localStorage.getItem; returns string or nil.
  • (js/local-set! k v) — localStorage.setItem; values must be strings (encode JSON or pr-str first).

Async

  • (js/fetch-text url callback) — fetch the URL as text; callback is invoked with the body string (or nil on failure). Cannot return the body synchronously because cljrs eval is re-entrant-unsafe.

Adding a new JS package

All builtins live in crates/cljrs-wasm/src/js_bridge.rs. Adding a new one is a four-line patch.

  1. Pick a web-sys / js-sys API. If it exists in web-sys already, enable the feature in crates/cljrs-wasm/Cargo.toml (each web-sys interface is feature-gated).
  2. Write the builtin. Match the existing pattern: 
    fn my_thing_fn(args: &[Value]) -> Result<Value> {
    let s = arg_str(args, 0, "js/my-thing")?;
    // ...call the API...
    Ok(Value::Nil)
}
  3. Register it. Add a bind(env, "my-thing", my_thing_fn); line inside install, in both the js and cljrs.js blocks (mirrored on purpose so qualified imports resolve).
  4. Round-trip JsValues with the newtype. JsValue is !Send + !Sync; cljrs Opaque.inner requires Send + Sync. Wrap in JsValueWrapper (the file's existing newtype, with a unsafe impl Send + Sync — sound on wasm32 since the target has no real threads).

Async APIs

Use wasm_bindgen_futures::spawn_local to drive Promises on the JS event loop, then eval::apply the stashed cljrs callback when the future resolves. Don't try to block the cljrs evaluator on a Promise — the interpreter is re-entrant-unsafe, and you'd clobber env state if a caller is mid-evaluation. fetch_text_fn is the canonical template.

Event closures

For one-shot listeners use Closure::wrap + Closure::forget as in on_fn. The closure lifetime becomes "page lifetime", which matches typical usage. If you need detach, return a disposer handle (out of scope for the starter API, easy enough to add per-call).
