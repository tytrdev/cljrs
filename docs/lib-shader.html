Shader stdlib

A userland "standard library" for cljrs defn-gpu-pixel kernels, written entirely as defmacros (demo_gpu/stdlib.clj). Zero compiler changes: each helper expands inline via syntax-quote so the GPU emitter sees a single flat scalar expression tree, exactly as if you had written it by hand. The DSL is scalar-only at the user level — helpers that "want" to return vec3 either expose a channel selector or expand to a let block you destructure.

Use it by loading the file at the top of a kernel source: (load-file "demo_gpu/stdlib.clj"). The gpu-compile binary evaluates non-kernel forms before extracting the kernel body, so the macros are registered in time for macroexpand-all.

GPU compilation needs a real GPU pipeline (WGSL emitter, native driver or WebGPU). The runnable cljrs-wasm Repl can macro-expand these helpers but cannot execute the resulting kernel. The examples below show the macro source and a comment-form sketching what the expanded code looks like; for live kernels, see GPU.

Color packing

  • (pack-rgb r g b) — clamp three f32 channels into [0,1], scale to 8-bit, return one packed u32 in 0x00RRGGBB form.
  • Every kernel ends with this; factoring it out saves four lines at the bottom.

Noise

  • (hash2 ix iy) — xxHash-flavored integer hash to f32 in [0,1). Expects integer cell coords.
  • (noise2 xf yf) — 2D value noise with smoothstep weights. f32 coords; period 1.
  • (fbm2 xf yf) — five-octave fractional-Brownian-motion sum, normalized to ~[0,1].
  • Gotcha: every literal fits in i32 so the emitter's integer path is happy. Don't replace them with f32 constants.

SDF primitives

  • (sd-sphere px py pz cx cy cz r) — signed distance to a sphere of radius r centered at (cx cy cz).
  • (sd-box px py pz cx cy cz hx hy hz) — axis-aligned box, half-extents (hx hy hz).
  • (sd-plane px py pz y0) — infinite Y=y0 plane. Negative below, positive above.
  • (sd-torus px py pz cx cy cz R r) — Y-axis torus, major R, minor r.
  • All primitives return one f32 (signed distance), so they compose freely with the combinators below.

SDF combinators

  • (sd-union a b) — min.
  • (sd-intersect a b) — max.
  • (sd-subtract a b) — carve b out of a: (max a (- b)).
  • (sd-smooth-union a b k) — IQ polynomial smooth-min with blend radius k; k → 0 recovers sd-union.

Tonemapping

  • (tone-reinhard v) — single-channel v / (1 + v). Keeps highlights from clipping when summing several light contributions.

FBM, sketched

The five-octave sum below shows the structure fbm2 expands to. On the GPU, each noise2 is itself the smoothstep-weighted bilerp built from hash2. 

(defmacro fbm2 [xf yf]
  `(let [n1# (noise2 ~xf ~yf)
         n2# (noise2 (* ~xf 2.07) (* ~yf 2.07))
         n3# (noise2 (* ~xf 4.13) (* ~yf 4.13))
         n4# (noise2 (* ~xf 8.19) (* ~yf 8.19))
         n5# (noise2 (* ~xf 16.3) (* ~yf 16.3))
         s#  (+ (+ (* n1# 0.5) (* n2# 0.25))
                (+ (* n3# 0.125) (+ (* n4# 0.0625) (* n5# 0.03125))))]
     (/ s# 0.96875)))

Each octave doubles the frequency (with a slight irrational offset to break grid alignment) and halves the amplitude. The final divide by 0.96875 renormalizes the geometric sum.
