Sequencer — live coded

In the spirit of TidalCycles and Sonic Pi: rhythms and melodies are vectors of data in Clojure. Edit the source, the grid and the drum machine update. Companion to the subtractive synth demo.

A 16-step, 4-track step sequencer. Three percussion tracks (kick / snare / hat) plus a melody line whose pitches come from the cljrs.music library — try changing the scale from :minor to :dorian or :pentatonic and watch the melody re-harmonize on the next bar. Web Audio handles sample-accurate scheduling; the cljrs runtime owns the score.

Hold Alt (or Ctrl) and drag any number — try BPM — to scrub it live. Press Cmd/Ctrl+Enter to apply immediately.

What's happening

  • The Clojure source defines four 16-element vectors and a BPM. applySource re-reads them with repl.eval_floats on every edit (300 ms debounce) — the JS sequencer reads the freshest pattern at the start of every step.
  • The melody line uses (cljrs.music/scale 60 :minor) to map step indices into MIDI notes, demonstrating data-driven harmony from the cljrs.music library.
  • Web Audio scheduling: a setInterval-driven lookahead walks the next ~100 ms of steps and queues drum hits / oscillator notes with audioCtx.currentTime, the standard Chris Wilson scheduling pattern. The cljrs interpreter is fast enough for per-frame logic but sample-accurate timing belongs in the audio thread.
  • The grid you see is a view of the Clojure data, not a backing store — there's no checkbox state in JS, only what the source defines.
