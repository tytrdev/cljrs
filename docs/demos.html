Live-coded demos

21 graphical kernels, all written in cljrs and JIT-compiled to native code via MLIR. Each video was rendered by cargo run --release --bin record from the same source tree you can open and edit — the recorder is part of the project, not a separate pipeline.

Running natively, each demo updates at 60+ fps with the editor open. Parameter sliders, time scrubbing, and reload-on-save are all wired up in the demo binary (cargo run --release --features demo --bin demo -- demo/fractal.clj).

GPU kernels (defn-gpu-pixel)

These kernels compile from cljrs source to WGSL and run on the GPU via wgpu — Metal on macOS, Vulkan on Linux, DX12 on Windows, and WebGPU in supported browsers. Rendered 252–517 fps on an integrated Apple M3 at 640×360.

Run locally with: cargo run --release --features gpu-demo --bin gpu-demo -- demo_gpu/plasma.clj. Edit the .clj file while the window is open — the kernel recompiles on save (same Bret-Victor loop as CPU).

CPU kernels (defn-native)

21 kernels below, each written in cljrs and JIT-compiled to native code via MLIR + LLVM. Rendered via bin/record.

How the kernel ABI works

Each demo exports a single function with a fixed signature: 

(defn-native render-pixel ^i64
  [^i64 px ^i64 py ^i64 frame ^i64 t-ms
   ^i64 s0 ^i64 s1 ^i64 s2 ^i64 s3]
  ...body... )

Returns an i64 packed as 0xRRGGBB. The host calls it once per pixel per frame in parallel via rayon. s0 through s3 are the UI slider values (0..1000). Time scrubbing replaces t-ms with the scrubbed value when the user pauses.

Building your own

echo "(defn-native render-pixel ^i64
  [^i64 px ^i64 py ^i64 frame ^i64 t-ms
   ^i64 s0 ^i64 s1 ^i64 s2 ^i64 s3]
  (+ (* px 257) py))" > demo/stripes.clj

cargo run --release --features demo --bin demo -- demo/stripes.clj

Edit the file while the window's open — the kernel recompiles on save. No restart.

Recording your own videos

cargo run --release --features demo --bin record -- \
  --kernel demo/fractal.clj \
  --seconds 6 --fps 60 --size 960x540 \
  --out out/fractal.mp4

Pipes raw RGBA frames to ffmpeg. The videos on this page were produced with this command.
