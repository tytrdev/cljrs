Roadmap

Rough shape. Dates are aspirational, priorities are stable.

Where we are

  • Tree-walker, reader, macro system — stable.
  • MLIR-JIT native path for type-hinted fns — stable, benchmarked.
  • Persistent collections, destructuring, threading/control macros — stable.
  • Atoms, try/catch, ~125 builtins — stable.
  • Live-coded graphics demos with editor, sliders, time scrubbing — stable.
  • Headless recorder (bin/record) for making demo videos — stable.
  • WASM bundle (tree-walker) + docs-site REPL — stable.

Next

  1. GPU kernel DSL (native). defn-gpu macro expanding to MLIR gpu dialect, lowered to Metal on macOS, Vulkan/SPIR-V elsewhere. Dispatch via wgpu. Same buffer ABI as defn-native — the f64-buf type + buf-get/buf-set primitives are already there, waiting.
  2. GPU kernel DSL (web). Same kernel emitted as WGSL, dispatched via wgpu inside the WASM bundle. Unlocks real GPU compute in the docs-site demos.
  3. GPU benchmarks. Match the CPU benchmarks page: matmul, elementwise, reductions, stencil, N-body — cljrs-gpu vs numpy, pytorch, jax, julia-CUDA. Same methodology rigor (sources published, biases listed).

Clojure coverage wishlist

  • defprotocol / deftype / defrecord.
  • Proper namespaces with :refer, :as, isolation.
  • Lazy-seqs & transducers.
  • Auto-gensym (x#).
  • Regex literals.
  • Ratios & bigint.

Non-goals

  • JVM interop. We're not shipping import for Java classes.
  • Bit-for-bit Clojure compatibility. We match semantics where it's cheap and diverge where the JVM baggage isn't worth it.
  • A package manager. load-file + require are the story until someone wants more.
