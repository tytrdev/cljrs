cljrs.ml

A small, deliberate CPU autograd built on dense f32 matrices, exposed to cljrs through opaque tensor handles (Value::Opaque{tag = "ml/tensor"}). The crate lives at crates/cljrs-ml; it's the canonical example of "build a numeric library in Rust, expose it as cljrs builtins."

Design goals: enough surface area to train a small MLP from a Repl, no runtime dependencies (no BLAS, no CUDA), deterministic seeds so a session reruns identically. Tensors are 2D row-major; use shape (1 n) for vectors. Backward is reverse-mode autodiff over the Tensor graph; (ml/sgd-step! params lr) mutates each (ml/param ...) in place.

Construction

  • (ml/tensor data) — wrap a 1-D or 2-D vector of numbers; shape inferred.
  • (ml/tensor rows cols data) — explicit shape; data is a flat or nested vector totalling rows*cols numbers.
  • (ml/param data) · (ml/param rows cols data) — same as tensor but flagged as a learnable parameter. Only params get updated by sgd-step!.
  • (ml/randn rows cols) · (ml/randn rows cols stddev) — Box-Muller Gaussian init from a deterministic xorshift RNG. Always returns a param.
  • (ml/zeros rows cols) — zero-initialized param.

Init helpers

  • (ml/xavier rows cols) — Glorot init: stddev = √(2 / (in + out)). Sane default for tanh/sigmoid layers.
  • (ml/kaiming rows cols) — He init: stddev = √(2 / fan_in). Sane default for relu/gelu layers.

Ops (build the graph)

  • (ml/matmul a b) — matrix multiply; inner dims must match.
  • (ml/matmul-gpu a b) — same shape contract; dispatches a wgpu compute kernel on native, falls back to CPU on wasm with a one-time warning. Wired into the autograd graph so backward still flows.
  • (ml/add a b) · (ml/sub a b) — element-wise; shapes must match.
  • (ml/add-bias a b) — broadcast a (1 cols) bias across every row of a.
  • (ml/relu x) · (ml/tanh x) · (ml/sigmoid x) · (ml/gelu x) — element-wise nonlinearities.
  • (ml/softmax x) — row-wise softmax (numerically stabilized). Rows sum to 1.
  • (ml/conv1d-valid input kernel) — 1-D valid convolution. Both args must be (1 N); output is (1 N - K + 1).

Losses

  • (ml/mse pred target) — mean squared error.
  • (ml/mae pred target) — mean absolute error; less sensitive to outliers.
  • (ml/cross-entropy probs targets) — categorical cross-entropy. Pair with ml/softmax on the prediction side and ml/one-hot on the target side.
  • All three reduce to a (1 1) tensor — feed straight into backward!.

Train

  • (ml/backward! loss) — reverse-mode autodiff over the graph rooted at loss. Accumulates grads on every reachable param.
  • (ml/sgd-step! params lr) — for each param: w <- w - lr*grad, then zero the grad.
  • (ml/adam-step! params lr) · (ml/adam-step! params lr beta1 beta2 eps) — Adam optimizer. Defaults: 0.9, 0.999, 1e-8.
  • (ml/rmsprop-step! params lr) · (ml/rmsprop-step! params lr decay eps) — RMSProp; defaults 0.9, 1e-8.
  • (ml/reset-optim!) — wipe Adam/RMSProp moment buffers (useful between unrelated training runs in one session).
  • (ml/set-data! tensor data) — overwrite a tensor's buffer in place. Use to stream new minibatch inputs without rebuilding the graph wrapper. Shape must match.

Inspect & post-process

  • (ml/scalar t) — first element as a number; convenient for losses.
  • (ml/tolist t) — flat vector of f32s (boxed as cljrs floats).
  • (ml/shape t) — [rows cols].
  • (ml/argmax t) — index of the max along each row. Single-row tensors return an Int; multi-row tensors return a vector of Ints.
  • (ml/one-hot k n) · (ml/one-hot [k0 k1 ...] n) — build a one-hot tensor of width n. Multi-index form returns one row per index.
  • (ml/normalize t) — z-score normalize the whole buffer; returns a fresh leaf (no autograd flow).
