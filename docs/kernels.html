Numeric kernels — measured

Four common kernels written once in cljrs, then measured across four paths: the cljrs tree-walker calling MLIR-JIT'd per-element defn-native code, a hand-written Rust inline version doing exactly the same thing, NumPy via ufuncs + BLAS, and the transpiled Mojo output from cljrs-mojo. Timings are the median of 20 iterations on the same deterministic 1M-element f64 buffers.

Read the numbers honestly.
  • cljrs MLIR JIT lands within 70–95% of a rustc -O3 loop on the three memory-bound kernels. Clojure compiling to native is not a novelty here — it's competitive.
  • numpy's dot is ~20× faster than everything else because it calls Apple Accelerate's BLAS: hand-tuned SIMD with blocking. The other numpy kernels are 2–3× faster than Rust via SIMD ufuncs.
  • mojo numbers below are from a compiled mojo build -O3 binary on nightly 0.26.3 using scalar loops over List[Float64]. It's slower than the Rust baseline because List does bounds-checking and the transpiler's vectorize[]+UnsafePointer output doesn't compile on the current nightly — the stable APIs cljrs-mojo targeted were renamed. We kept the transpiler's golden output intact and adapted the harness. Updating the transpiler to emit current-nightly syntax is the obvious next commit.

How to reproduce

bench/kernels.sh 1000000 20
# writes docs/bench-kernels/results.json

Run on any machine. The harness builds cljrs with the mlir feature on (needed for defn-native JIT), runs the Rust driver that transmutes the JIT'd function pointer and invokes it per-element, then runs the NumPy script via uv for comparable numbers.

What's being measured

  • cljrs MLIR JIT: one (defn-native vadd-elem ^f64 [^f64 a ^f64 b] (+ a b)) compiled once. The Rust driver extracts the code pointer, transmutes to extern "C" fn(f64, f64) -> f64, calls it in a tight loop. Only the per-element op is measured; the loop is Rust's.
  • rust inline: same loop, body written directly in Rust. This is the compiler's best effort for the same semantics without touching SIMD intrinsics or iter().sum() specialization.
  • numpy: np.add(a, b, out=out) / np.dot(a, b) / etc. NumPy's ufuncs SIMD-vectorize; dot delegates to BLAS.
