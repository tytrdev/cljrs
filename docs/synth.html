Synth — live coded

Inspired by Sonic Pi, Overtone, and TidalCycles — live-coded music as a first-class way to explore sound.

A single-voice subtractive synth whose patch parameters are authored in Clojure. The DSP graph (oscillator → filter → gain → output) is a fixed Web Audio graph in the browser; every edit to the source re-reads the params and pushes them onto the audio nodes with setValueAtTime. Hold Alt (or Ctrl) and drag any number to scrub it live while the note plays. The canvas on the left is an oscilloscope fed from an AnalyserNode.

Why this split? The cljrs tree-walker is fast enough for per-frame game logic but not for 48 kHz per-sample DSP. Patch-level params through Clojure, sample-level DSP in Web Audio — the call_floats WASM hook is also in place for a future all-Clojure buffer-render path.

What's happening

  • Each def at the top of the source is a patch knob. Editing one and pausing for ~400 ms triggers applySource, which re-reads the knobs with repl.eval_floats and schedules new values on the matching Web Audio nodes.
  • WAVE-TYPE is a keyword — JS reads its name via repl.eval, maps it to an OscillatorNode.type.
  • Trigger re-runs the ADSR envelope by calling (patch) on the cljrs side, which returns the full flat param vector the JS graph consumes.
  • The new repl.call_floats(name, Float32Array) WASM method is what a full DSP-in-Clojure port would use — a per-callback fn dispatch that skips the reader.
