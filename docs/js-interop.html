JS interop — DOM, fetch, localStorage from cljrs

The cljrs runtime lives entirely in WASM. The cljrs.js namespace exposes browser primitives — console.log, document.getElementById, addEventListener, localStorage, fetch — as ordinary cljrs builtins. DOM elements round-trip as opaque handles.

    • (js/on! el "click" handler) — handler is a 0-arg cljrs fn
    • (js/local-get k) · (js/local-set! k s)
    • (js/fetch-text url callback) — callback receives the body string (or nil on error)
