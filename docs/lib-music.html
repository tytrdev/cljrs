cljrs.music

A small music-theory toolkit, authored entirely in cljrs and bundled into the prelude (see src/cljrs_music.clj). All functions are pure data → data — no audio, no scheduling. The Sequencer page reads the vectors out of these helpers and plays them through Web Audio.

MIDI & tuning

  • (cljrs.music/midi->hz n) — MIDI int → Hz, A4 = 69 = 440.
  • (cljrs.music/hz->midi f) — inverse; returns f64 (no rounding).

Note keywords

  • (cljrs.music/note kw) — parse a keyword like :c4, :a#3, :eb5 into a MIDI int.
  • Form: letter, optional # or b, then octave digits. Bare :c defaults to octave 4.
  • Throws ex-info on unknown pitch class.

Scales

  • (cljrs.music/scale root mode) — return a vector of MIDI ints. root is a MIDI int; mode is a keyword.
  • Modes: :major :minor :aeolian :dorian :phrygian :lydian :mixolydian :locrian :pentatonic :blues.
  • Includes the octave (8 notes for 7-note scales) so loops can wrap cleanly.

Chords

  • (cljrs.music/chord root quality) — root + interval table → MIDI vector.
  • Qualities: :maj :min :dim :aug :sus2 :sus4 :7 :maj7 :min7 :dim7.

Progressions

  • (cljrs.music/progression key qualities) — diatonic-major degree resolution. qualities is a vector of [degree quality] pairs; degrees are 1–7.
  • Returns a vector of chords (each itself a vector of MIDI ints).

Manipulation

  • (cljrs.music/transpose notes semitones) — add an offset to every note.
  • (cljrs.music/arpeggiate chord-notes pattern) — reorder a chord into a melodic pattern. Patterns: :up :down :updown :random. :random is deterministic (cljrs has no rand yet) — it cycles a fixed permutation.

Audio

Synthesis lives in the demo pages (Synth, Sequencer) — the Web Audio glue is JS, not cljrs. The cljrs side just produces frequency vectors and step patterns; the JS side schedules OscillatorNodes on an AudioContext. If you'd like a richer audio API exposed under a cljrs.audio namespace, the right pattern is the cljrs.js bridge — wrap AudioContext as an opaque, expose a few mutators.
