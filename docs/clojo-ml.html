Clojo trained a real model

A logistic regression classifier trained end-to-end through the cljrs → Mojo pipeline. Target: Wisconsin breast cancer (569 biopsy measurements × 30 features, binary malignant/benign). Same split scikit-learn uses in its regression tutorial: 80/20 stratified, random_state=42. The kernels are five cljrs forms; everything else (data load, epoch loop, metrics) is plain Mojo calling into them.

How to reproduce

bench/clojo_logreg/run.sh
# transpiles kernels.clj → kernels.mojo.{readable,optimized,max}
# runs prepare_data.py (downloads dataset, emits f64 binaries)
# mojo build -O3 kernels + train harness, runs
# writes docs/clojo-ml/results.json

Requires the mojo binary on PATH — install via uv and Modular's nightly wheel index (see the numeric kernels page for setup). The transpiled Mojo works on nightly 0.26.3; the transpiler follows the active syntax rather than a pinned version.

What this shows and what it doesn't

  • Shows: cljrs-mojo generates Mojo that compiles with mojo build -O3, links against real stdlib types (UnsafePointer, SIMD), and executes a real training loop that converges on real data to a model that matches (or beats) a hand-written scikit-learn baseline.
  • Doesn't show: deep learning. No autograd, no matmul primitive, no batching. This is logistic regression because it decomposes cleanly into dot + elementwise, which is exactly the scope of kernels the transpiler currently supports well.
  • The path forward: add matmul and conv as first-class kernel forms. Once those exist, MLPs fall out naturally.
