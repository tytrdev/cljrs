Clojure, reimplemented for performance.

cljrs is a Clojure dialect built from scratch in Rust. The reader, evaluator, macro system, and persistent collections are a plain tree-walker. The performance-critical path compiles type-hinted fns straight to native machine code via MLIR + LLVM — no JVM, no startup tax.

The pitch in one snippet

;; Tree-walker fn — easy, dynamic, like any Lisp.
(defn avg [xs] (/ (reduce + xs) (count xs)))

;; Type-hinted fn — compiled to native machine code. Same syntax.
(defn-native fib ^i64 [^i64 n]
  (if (< n 2) n (+ (fib (- n 1)) (fib (- n 2)))))

(fib 35)  ; runs at native-Rust speeds

Why

Clojure is a beautiful language bolted to a startup-hostile runtime. cljrs drops the JVM entirely: persistent collections via imbl, macros and reader in pure Rust, and a JIT backend that emits the same MLIR / LLVM pipeline production compilers use. The goal is Clojure that runs fast enough for games, numerical kernels, and eventually GPU compute — native AND in the browser via WebGPU + WASM.

Status

  • 2.4–13× faster than Clojure on JVM on compute micro-benchmarks. See Benchmarks.
  • Live-coded graphical demos with parameter sliders and time scrubbing. See Demos.
  • GPU kernel DSL and cross-platform (native + WebGPU) dispatch are on the Roadmap.
