cljrs.ui

A tiny hiccup-style UI library, authored in cljrs and bundled into the prelude. Two halves:

  1. A pure render-html that converts hiccup vectors to an HTML string. Used at build time by src/bin/prerender.rs so docs pages ship SEO-visible HTML before any JS runs.
  2. mount! / hydrate! / reactive! — wasm-only helpers that dispatch to host builtins (__ui-mount, __ui-hydrate) installed by crates/cljrs-wasm/src/ui_bridge.rs. Backed by a small Preact runtime; the cljrs side just produces hiccup.

Hiccup shape

[:div {:class "x"}
  [:button {:on-click f} "go"]
  [:span "hi"]]
  • First element is a keyword tag.
  • Optional second element is a props map — including :on-click, :on-input, etc., which take cljrs fns.
  • Remaining elements are children. Strings/numbers are stringified+escaped; nil children are skipped; nested vectors recurse.

Pure render

  • (cljrs.ui/h tag props & children) — convenience constructor; identical to writing the literal vector.
  • (cljrs.ui/render-html hv) — hiccup → HTML string. Pure, build-time-safe, no DOM access. Escapes attrs and text. Skips event handlers (they only matter at hydrate time).

Mount & hydrate (wasm only)

  • (cljrs.ui/mount! root-id hiccup) — diff-replace the contents of #root-id with the rendered hiccup. Wires event handlers.
  • (cljrs.ui/hydrate! root-id hiccup) — attach event handlers to existing pre-rendered DOM without replacing nodes. Pair with render-html at build time for zero-flash first paint.
  • (cljrs.ui/reactive! root-id state-atom view-fn) — initial paint + return a 0-arg re-render thunk you call after each state mutation. cljrs has no add-watch, so the thunk pattern keeps this layer leaf-shaped.

Build-time prerender

src/bin/prerender.rs evaluates a .cljrs file in a native cljrs and calls its (prerender) function, which must return hiccup. The binary stringifies the result through render-html and docs/build-ui.sh injects it into the page between <!--ssr--> markers. At page load, the same module's (boot) runs hydrate! on the already-painted DOM. See ui-demo for the canonical end-to-end example.
