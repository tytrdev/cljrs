ML — live coded autograd

A tiny MLP fits a noisy sin(x) in your browser. The model definition, hyperparameters, and one training step are all authored in Clojure; only forward + backward + SGD live in Rust.

Edit HIDDEN-SIZE, LEARNING-RATE, the activation, or the target function — pause for ~400 ms and the page rebuilds the model with new weights. Hold Alt (or Ctrl) and drag any number to scrub it live. Weights live behind defonce so editing the source while training continues only resets them when you change shape.

Phase 1: CPU autograd. Reverse-mode over dense f32 buffers in a new cljrs-ml crate, exposed to cljrs as builtins (ml/tensor, ml/matmul, ml/relu, ml/mse, ml/backward!, ml/sgd-step!, …). The cljrs GPU path is per-pixel right now — wrong shape for matmuls — so a GPU mat-mul kernel is left as phase 2.

What's happening

  • The Clojure source defines HIDDEN-SIZE, LEARNING-RATE, BATCH, the dataset, and the model fns. It also calls ml/randn / ml/zeros behind defonce so weights survive an edit.
  • Each frame the page calls (train-step!) which does a forward pass, a backward pass, and one SGD update. It returns the current loss; JS plots it.
  • (predict-grid) returns model outputs over a fixed grid of x values; JS draws it as a curve over the dataset.
  • If you change HIDDEN-SIZE the JS detects a shape mismatch on the next frame and re-applies the source, which re-initializes the weights.
